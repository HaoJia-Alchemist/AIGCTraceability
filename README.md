# AIGCTraceability

AIGC Image Traceability and Deepfake Detection Platform, providing multiple deep learning models for detecting and tracking AI-generated images.

## Introduction

This project aims to solve the traceability problem of AIGC (AI-Generated Content) images. It uses deep learning techniques to identify whether images are generated by specific AI models and supports tracking of generation sources. The project integrates various advanced pre-trained models and training strategies, suitable for academic research and practical applications.

## Key Features

- **Multi-Model Support**: Integrates CLIP ViT, EfficientNet, ResNet, Xception, Effort, and other pre-trained models
- **Flexible Training Strategies**: Supports Triplet Loss, Center Loss, Prompt Learning, Caption Decoupling, and other training methods
- **Dataset Generation**: Provides complete AIGC image dataset generation tools, supporting multiple mainstream generation models
- **Multi-GPU Training**: Distributed training support based on Accelerate framework
- **Rich Data Augmentation**: Offers various data augmentation methods including image flipping, rotation, blurring, brightness adjustment, etc.

## Usage

### Model Training

#### Single GPU Training
```bash
cd DeepfakeTraceability
python train.py --config ./config/configs_dir/clip_vit_b16.yaml --train_config_file ./config/train_config.yaml
```

#### Multi-GPU Training
```bash
cd DeepfakeTraceability
bash scripts/start_train.sh
```

Supported model configuration files:
- `clip_vit_b16.yaml` - CLIP ViT-B16 model
- `clip_vit_l14.yaml` - CLIP ViT-L14 model
- `efficientnet.yaml` - EfficientNet model
- `resnet50.yaml` - ResNet50 model
- `xception.yaml` - Xception model
- `effort.yaml` - Effort model
- `prompt_caption_decoupling.yaml` - Prompt Caption Decoupling model
- `dual_data_alignment.yaml` - Dual Data Alignment model

### Model Testing

```bash
cd DeepfakeTraceability
python test.py --config ./config/configs_dir/clip_vit_b16.yaml --train_config_file ./config/train_config.yaml
```

Or use the script:
```bash
bash scripts/start_test.sh
```

### Configuration Guide

#### Model Configuration
Configuration files are located in the `config/configs_dir/` directory. Main parameters include:

- `model.name`: Model name
- `model.feat_dim`: Feature dimension
- `model.metric_loss_type`: Loss function type (triplet/center/supcon)
- `train.max_epoch`: Number of training epochs
- `dataset.train_batch_size`: Training batch size
- `solver.base_lr`: Learning rate

#### Training Configuration
Main configuration file `train_config.yaml`:

- `gpu_ids`: List of GPU IDs to use
- `dataset.root_dir`: Dataset root directory
- `dataset.dataset_json_file`: Dataset JSON file path
- `logging.log_dir`: Log output directory

## FAQ

**Q: How to use my own dataset?**

A: You need to prepare a JSON format dataset annotation file containing three parts: train, query, and gallery, and set the correct paths in the configuration file.

**Q: How to adjust training parameters?**

A: You can directly modify the corresponding YAML configuration file, or override via command line arguments:
```bash
python train.py --config ./config/configs_dir/clip_vit_b16.yaml --opt solver.base_lr=0.001
```

**Q: How to resume training?**

A: Set the resume parameter in train_config.yaml to the checkpoint path:
```yaml
resume: /path/to/checkpoint
resume_epoch: 10
```
